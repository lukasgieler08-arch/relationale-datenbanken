Teil B: Referentielle Integrität

Kernidee

Ein Fremdschlüssel darf nur auf einen vorhandenen Primärschlüssel verweisen (oder auf NULL, wenn erlaubt).

Beispiel Lernlabor

  • SCHUELER(SchuelerID PK, Name)
  • WORKSHOP(WorkshopID PK, Titel)
  • TEILNAHME(TeilnahmeID PK, SchuelerID FK, WorkshopID FK)

Eine Teilnahme ist nur gültig, wenn Schüler und Workshop existieren.

Integritätsaktionen

  • RESTRICT/NO ACTION
  • CASCADE
  • SET NULL
Merksatz: Beziehungen brauchen Regeln, sonst entstehen verwaiste Datensätze.
Typischer Fehler: FK und PK mit unpassenden Datentypen modellieren.
