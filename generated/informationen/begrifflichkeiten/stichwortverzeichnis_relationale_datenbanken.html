Begriff Definition Beispiel
1. Normalform (1NF)In jeder Zelle steht genau ein einzelner Wert.Keine Listenwerte in einer Spalte.
2. Normalform (2NF)Nichtschlüsselattribute hängen vom gesamten Primärschlüssel ab.Bei (SchülerID, KursID) steht Schülername nicht mit in derselben Tabelle.
3. Normalform (3NF)Keine transitiven Abhängigkeiten zwischen Nichtschlüsselattributen.Raumname nicht direkt in KURS, sondern in RAUM.
AggregatfunktionFasst viele Zeilen zu einem Ergebnis zusammen.COUNT(*), SUM(preis), AVG(punkte).
ALTER TABLEÄndert die Struktur einer bestehenden Tabelle.Spalte hinzufügen mit ALTER TABLE.
ANDAlle Bedingungen müssen wahr sein.WHERE ort='Stuttgart' AND aktiv=1
AnomalieUnerwünschter Nebeneffekt durch schlechte Struktur.Beim Löschen gehen fachlich wichtige Infos verloren.
AS (Alias)Vergibt Kurznamen für Tabellen oder Spalten.SELECT k.name AS kundenname FROM kunden k
AttributEigenschaft einer Entität.Geburtsdatum, Nachname, Preis.
AtomaritätEin Feld enthält nur einen unteilbaren Einzelwert.Keine zwei Telefonnummern in einer Zelle.
BETWEENPrüft, ob ein Wert in einem Bereich liegt.WHERE preis BETWEEN 10 AND 50
BeziehungZusammenhang zwischen Entitäten.Ein Kunde hat viele Bestellungen.
CHECKRegel für erlaubte Werte in einer Spalte.CHECK (punkte >= 0)
COUNTZählt Datensätze oder Werte.SELECT COUNT(*) FROM bestellung
CREATE DATABASEErstellt eine neue Datenbank.CREATE DATABASE schulshop_db;
CREATE TABLEErstellt eine neue Tabelle.CREATE TABLE kunde (...)
DatenkonsistenzDaten sind widerspruchsfrei und regelkonform.Keine Bestellung mit unbekannter Kunde-ID.
Datensatz (Tupel)Eine Zeile in einer Tabelle.Eine konkrete Ausleihe mit Datum.
DatentypLegt fest, welche Art von Daten gespeichert wird.INT, DATE, VARCHAR.
DELETELöscht Datensätze aus einer Tabelle.DELETE FROM bestellung WHERE id=17
DISTINCTEntfernt Duplikate in Abfrageergebnissen.SELECT DISTINCT ort FROM kunden
DDLSQL-Befehle für Datenbankstruktur.CREATE, ALTER, DROP.
DMLSQL-Befehle für Dateninhalte.INSERT, UPDATE, DELETE.
EinfügeanomalieDaten können nur unvollständig oder künstlich eingefügt werden.Kurs erst anlegbar, wenn schon Teilnehmer existiert.
EntitätKonkretes Objekt der realen Welt.Eine Kundin oder ein Fahrrad.
EntitätstypKlasse ähnlicher Entitäten (oft eine Tabelle).KUNDE, PRODUKT, BUCHUNG.
ER-Modell / EERMModell zur fachlichen Datenbankplanung.Vor SQL wird ein EERM in Workbench gezeichnet.
Fremdschlüssel (FOREIGN KEY)Verweis auf Primärschlüssel einer anderen Tabelle.bestellung.kunde_id → kunde.kunde_id
FROMLegt die Ausgangstabelle einer Abfrage fest.SELECT * FROM kunde
GROUP BYBildet Gruppen für Auswertungen.GROUP BY stadt
HAVINGFiltert Gruppen nach GROUP BY.HAVING COUNT(*) >= 2
INPrüft auf Mitgliedschaft in einer Werteliste.WHERE stadt IN ('Ulm','Heilbronn')
IndexBeschleunigt das Finden von Datensätzen.Index auf kundennummer.
INNER JOINNur Treffer in beiden Tabellen.Kunden mit vorhandenen Bestellungen.
INSERTFügt neue Datensätze ein.INSERT INTO kunde (...) VALUES (...)
IntegritätsregelRegel zur Sicherung gültiger Daten.Primärschlüssel darf nicht NULL sein.
IS NULLPrüft auf fehlende Werte.WHERE lieferdatum IS NULL
JOINVerbindet Tabellen über Schlüsselbeziehungen.kunde JOIN bestellung ON ...
KardinalitätAnzahl möglicher Zuordnungen zwischen Entitäten.1:1, 1:n, n:m.
KandidatenschlüsselMöglicher eindeutiger Schlüssel einer Tabelle.E-Mail kann eindeutig sein.
LEFT JOINAlle Zeilen links bleiben erhalten.Alle Kunden, auch ohne Bestellung.
LIKEMustervergleich in Texten.WHERE name LIKE 'Sch%'
LöschanomalieBeim Löschen verschwinden ungewollt weitere Informationen.Letzte Ausleihe gelöscht, Stamminfo geht mit verloren.
MAXGrößter Wert.MAX(preis)
MINKleinster Wert.MIN(preis)
n:m-BeziehungViele Entitäten sind mit vielen anderen verknüpft.Viele Schüler besuchen viele Kurse.
NormalisierungSchrittweises Verbessern zur Redundanzvermeidung.Von Sammel-Tabelle zu 3NF-Modell.
NOTNegiert eine Bedingung.WHERE NOT status='inaktiv'
NOT NULLWert ist verpflichtend.name VARCHAR(100) NOT NULL
ONDefiniert Join-Bedingung.ON b.kunde_id = k.kunde_id
ORMindestens eine Bedingung muss wahr sein.WHERE ort='Freiburg' OR ort='Karlsruhe'
ORDER BYSortiert Abfrageergebnisse.ORDER BY nachname ASC
Primärschlüssel (PRIMARY KEY)Eindeutiger Identifikator eines Datensatzes.kunde_id identifiziert jeden Kunden genau einmal.
ProjektionAuswahl von Spalten.SELECT name, stadt FROM kunde
RedundanzMehrfachspeicherung gleicher Information.Kundename steht unnötig in vielen Zeilen.
Referentielle IntegritätFremdschlüssel dürfen nur auf vorhandene Primärschlüssel zeigen.Bestellung mit Kunde-ID nur, wenn Kunde existiert.
RelationTabellenartige Struktur im relationalen Modell.Die Tabelle KUNDE ist eine Relation.
SchemaGesamtstruktur einer Datenbank.Tabellen, Spalten, Schlüssel und Beziehungen.
SelektionAuswahl von Zeilen über Bedingungen.SELECT * FROM kunde WHERE ort='Mannheim'
SELECTLiest Daten aus Tabellen.SELECT name FROM kunde
SQLStandardsprache für relationale Datenbanken.Mit SQL werden Daten erstellt, geändert und ausgewertet.
Subquery (Unterabfrage)Abfrage innerhalb einer anderen Abfrage.WHERE preis > (SELECT AVG(preis) FROM produkt)
SUMSummiert Zahlenwerte.SUM(umsatz)
TabelleStruktur aus Spalten und Zeilen.Tabelle AUSLEIHE mit Start- und Endzeit.
Transitive AbhängigkeitIndirekte Abhängigkeit über ein Nichtschlüsselattribut.kurs_id → raum_id → raumname
UNIONVereint Ergebnisse zweier SELECT-Abfragen.SELECT ... UNION SELECT ...
UNIQUEWerte in einer Spalte müssen eindeutig sein.E-Mail nur einmal vorhanden.
UPDATEÄndert vorhandene Datensätze.UPDATE kunde SET ort='Ulm' WHERE kunde_id=3
WHEREFiltert Zeilen vor Gruppierung.WHERE aktiv=1
ÄnderungsanomalieEine fachliche Änderung muss mehrfach gepflegt werden.Telefonnummer muss in vielen Zeilen korrigiert werden.
